4.第4行


这是一行对变量（variable）的声明（declaration）

组分 含义
num 标识符（identifier），表示变量名
int 关键字（keyword），表示变量的数据类型为整数
; 声明的一部分

（What）声明变量做了两件事情：

  • 命名
  • 指定数据类型

（Why）在首次使用变量之前一定要先声明。

  • 不声明无法通过编译
  • 指定变量的数据类型，编译器才能在内存中为变量分配存储空间，计算机才能正确地存储、读取和解释数据
  • 有助于进行计划；变量放在一起，方便查找；便于发现程序中的变量名错误

（How）

  • 命名：
    • 可以使用小写字母、大写字母、数字和下划线，第1个字符必须是字母或下划线
    • 区分大小写
    • 要使用有意义的变量名，如果变量名无法清楚表达自身用途可在注释中说明
    • 名称长度有限制
  • 数据类型：将在后续章节中详细叙述

5.第5行

组分 含义
= 赋值运算符
num = 1 赋值表达式
num = 1; 赋值表达式语句

整个赋值表达式语句是把值 1 赋给名为num的变量。由于之前声明时，编译器在内存中为变量num预留了空间，因此执行这条语句时，就可以把值存储在之前预留的位置。

如何检查语法错误：

  • 编译前，发现和修正明显的错误
  • 看看编译器是否报错
    • 有时不用把编译器报告的所有错误逐一修正，仅修正第1条或前几处错误后，错误信息就会少很多
    • 报错的位置可能比真正的错误位置滞后一行

语义错误

语义错误的含义：源代码未违反 C 语言的规则，但是程序没有按照预期想法进行，导致结果错误。
举例：

  • n2;改为n3 = n2 n2;改为n3 = n2 n2;改为n3 = n2 n2;改为n3 = n2 2 * n;`

编译器不知道程序员的真正意图，无法检测语义错误。对于语义错误，可以跟随程序的执行，查看每一步的程序状态（program state，程序执行过程中，某个给定点上所有变量值的集合，是当前计算机状态的快照）。

定位语义错误的具体方法：

  • 自己模拟计算机逐步执行
  • 在关键点插入额外的printf语句，监视指定变量值的变化
  • 使用调试器（debugger）

勿随便使用关键字与保留标识符

  • 关键字，例如int、if：使用关键字不当会引起语法错误
  • 保留标识符（reserved identifier），例如printf：已被指定用途或保留使用权，如果用于表示其他意思会导致一些问题
