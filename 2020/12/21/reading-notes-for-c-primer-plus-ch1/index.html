C标准

C标准的发展

发布时间（年） 标准名称 发布者/发布机构 备注
1978 K&R C（经典C） 布莱恩·柯林汉和丹尼斯·理奇 出自两人合著的The C Programming 第1版，不过没有定义C库
1989/1990 C89或C90，即ANSI C或ISO C ANSI或ISO 首先由ANSI开发并发布，后被ISO采用
1994 C99 ANSI/ISO联合委员会（C9X委员会）
2011 C11

缩写的含义：

  • ANSI：，American National Standards Instiute，美国国家标准协会
  • ISO：International Organization for Standardization，国际标准化组织

查看Windows系统中GCC编译器所支持的C标准

  • 以管理员身份运行命令提示符（command prompt，cmd）
  • 执行命令：echo | gcc -dM -E - | find "__STDC_VERSION__"
  • 输出：#define __STDC_VERSION__ 201710L，说明是C18

编程机制

总体上，C编程是通过编译和链接，将源代码文件转换为可执行文件。有些系统中，必须分别编译和链接，另外一些系统中，编译器会自动启动链接器。

编译

编译器把源代码转换为中间代码（最普遍的形式是目标代码）。

链接

链接器将目标代码、启动代码（startup code）和库代码合并为可执行文件。

  • 库代码：例如，上述源代码中使用了库函数printf()，需要该库函数的机器语言代码
  • 启动代码：程序和操作系统之间的接口

编译器

  • 把高级语言翻译为机器语言。
  • 不同CPU制造商使用的指令系统和编码格式不同，意味着同一个机器语言程序可能无法在不同的CPU上运行。由于我们可以对特定类型的CPU找到与其相匹配的编译器，因此同一个高级语言程序可以在不同类型的CPU上使用。

GCC C 编译器

GCC 指 GNU 编译器集合（GNU Compiler Collection），是致力于开发自由软件的 GNU 项目的产品之一。GCC 中包含 C 编译器，称之为 GCC C 编译器。

GCC 是跨平台的，对不同的操作系统（包括UNIX、Lniux和Windows）都有相应的版本。经过持续不断的开发，其 C 编译器紧跟 C 标准的改动。
