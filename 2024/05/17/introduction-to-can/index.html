CAN入门
发布于：2024-05-17 14:58:56 访问：

控制器区域网络（Controller Area Nework，CAN）由博世公司开发，用于减少车辆中不断增长的微控制器和电子控制单元（Electronic Control Unit，ECU）的连接线数量。

基本术语

  • 车载诊断（On-board Diagnostics，OBD）：分为OBD-Ⅰ和OBD-Ⅱ
  • 差分对：使用两根传输线传输信号，将其中一根设置为高压，另一根设置为低压，测量两者间的电压差作为接收的信号。这两条传输线构成一个差分对。
  • ISO-TP：指 ISO 15765-2 协议，允许数据包在 CAN 上传输。CAN 协议本身对每个包的传输限制在 8 个字节，而 ISO-TP 使得更大的数据包可以在 CAN 上传输。

CAN

can-utils软件包

  1. 安装can-utils
    执行命令：
    sudo apt-get install can-utils
  2. 设置虚拟CAN网络
    由于缺少实际的设备，在这里使用虚拟 CAN 网络。

（1）加载 vcan 模块：sudo modprobe vcan

（2）使用dmesg命令显示消息，终端输出的最后一行为：

（3）使用ip link命令设置接口

sudo ip link add dev vcan0 type vcan
sudo ip link set up vcan0

（4）使用ifconfig命令确认 vcan0 接口存在：ifconfig vcan0
终端输出：

Tips: 如果提示Command 'ifconfig' not found, but can be installed with…，可以先用which ifconfig检查ifconfig是否已安装。若未安装，使用sudo apt install net-tools进行安装。
参考：Linux中ifconfig :command not found 找不到命令的解决方法_ifconfig未找到命令-CSDN博客

  1. 安装 can-isotp.ko模块
    can-isotp.ko 模块是一个 Linux 内核模块，后缀“ko”表示它是在 Linux 等类 Unix 操作系统中可以被加载的内核模块。通过名称可以知道它大概与 ISO-TP 协议有关，实际上它就是在 CAN 上为 ISO-TP 协议提供支持。

（1） 首先检查 can-isotp 模块是否已经被安装：稳定性得到验证以后（内核版本在 5.10 之后），被移到 Linux 内核的标准分支中。

  • 执行命令：modinfo can-isotp

（2）安装所需的依赖以及特定版本的 Linux 内核头文件

  • sudo apt-get install build-essential linux-headers-$(uname -r)

（3）下载：git clone https://github.com/hartkopp/can-isotp.git

（4）切换到can-isotp-master目录下进行编译：

  • cd can-isotp-master
  • make
  • sudo make modules_install
    报错：SSL error
    • 原因：Boot 模式是 UEFI，且 Secure Boot enabled
    • 解决：
      • 关闭 Secure Boot：选择 UEFI Firmware Settings，进入 Phoenix SecureCore Technology Setup，Security - Secure Boot，将 Enabled 切换为 Disabled，保存并退出
      • sudo modprobe can，成功以后终端无输出
    • 参考：
      • linux - insmod: ERROR: could not insert module HelloWorld.ko: Operation not permitted - Stack Overflow
      • missing step from installation instructions · Issue #25 · hartkopp/can-isotp (github.com)的“brandonros commented on Dec 18, 2020”

（5）使用 insmod 命令，将模块插入到内核中：sudo insmod ./net/can/can-isotp.ko，成功以后终端无输出

（6）使用demesg命令，查看是否有 isotp 协议
